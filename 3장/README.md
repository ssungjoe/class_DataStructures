# 3주차. 기초 데이터구조 (배열과 연결리스트)

<br>

## 1\. 데이터구조의 기본 재료

알고리즘과 데이터구조를 **건물 짓기**에 비유할 수 있다.

- **데이터구조**: 건물을 짓기 위한 재료, 자재, 그리고 뼈대(구조)에 해당한다.
- **알고리즘**: 완성된 건물에 들어가는 조명, 냉난방, 환기, 개폐 시스템 등 동작원리에 해당한다.

이 중에서도 가장 근간이 되는 '기본 재료'는 **배열(Array)**과 **연결리스트(Linked List)**다. 이들은 컴퓨터의 실제 구조와 매우 가까운 구체적이고 저수준(Low-level)의 개념이며, 앞으로 배우게 될 모든 고급 데이터구조(스택, 큐, 트리 등)를 만드는 필수 부품으로 사용된다.

<br>

## 2\. 배열 (Array)

**순차적인(연속된) 기억장소**에 할당된, **동일한 자료형**을 가진 유한 개수의 데이터 원소들의 모음이다.

### 2-1. 배열의 주요 용어와 표기법

- **배열명 ($V$)**: 배열 전체를 일컫는 대표 기호다.
- **배열 크기 ($N$)**: 원소를 저장하는 빈칸(셀)들의 총 개수다.
- **배열 첨자/인덱스 ($i$)**: 셀의 순위, 즉 배열 내에서의 상대적인 위치를 뜻한다.
  - 시작점: $LB$ (Lower Bound, 일반적으로 0)
  - 끝점: $UB$ (Upper Bound, 일반적으로 $N-1$)
- **배열 원소 ($V[i]$)**: 배열 $V$의 첨자 $i$번째 위치에 저장된 실제 데이터 값이다.
- **배열 표기법**: $V[LB..UB]$ 형태로 전체 범위를 표기하기도 한다.

### 2-2. 1차원 배열의 메모리 주소 계산

배열은 메모리상에 '연속적으로' 붙어있기 때문에, 시작 주소만 알면 산수 계산을 통해 특정 인덱스의 메모리 주소를 즉시 $O(1)$에 찾아낼 수 있다.

- **조건**: 배열의 시작 주소가 $x$이고, 각 원소가 차지하는 메모리 크기가 $c$바이트라고 가정하자. (인덱스는 0부터 시작)
- **주소 계산 공식**: $i$번째 원소 $V[i]$의 주소 = **$x + (i \times c)$**

### 2-3. 2차원 배열의 메모리 주소 계산 ($M \times N$ 크기)

2차원 배열(행렬)도 컴퓨터 메모리상에서는 결국 1줄(1차원)로 길게 펴져서 저장된다. 이때 펴는 방식에 따라 두 가지로 나뉜다. (시작 주소 $x$, 원소 크기 $c$)

1.  **행우선 (Row-major order)**
    - 가로줄(행)을 먼저 순서대로 메모리에 일렬로 깐다.
    - **C, C++, Java, Python** 등 대부분의 현대 언어가 사용하는 방식이다.
    - $(i, j)$ 위치의 주소 = **$x + (i \times N + j) \times c$** <br>_(※ 여기서 N은 총 열의 개수)_

    - 3차원 오프셋 공식: $(i \times M \times N) + (j \times N) + k$

2.  **열우선 (Column-major order)**
    - 세로줄(열)을 먼저 순서대로 메모리에 깔아버린다.
    - **Fortran, MATLAB** 같은 수학 연산 특화 언어에서 주로 사용한다.
    - $(i, j)$ 위치의 주소 = **$x + (j \times M + i) \times c$** <br>_(※ 여기서 M은 총 행의 개수)_

## 3. 연결리스트 (Linked List)

배열과 달리 메모리 공간상에 **비연속적(흩어져서)**으로 저장되며, 각 데이터가 다음 데이터의 주소를 가리키는 **포인터(링크)**로 연결된 데이터구조다.

### 3-1. 연결리스트의 핵심 특징

- **가변적인 길이**: 고정된 크기를 갖는 배열과 달리, 실행 중(Dynamic)에 메모리가 허용하는 한 무한히 크기가 늘어나거나 줄어들 수 있다.
- **접근 시간 ($O(n)$)**: 배열처럼 인덱스 계산으로 한 번에 찾아갈 수 없다. 무조건 맨 처음 노드부터 링크를 타고 순차적으로 따라가야 하므로 탐색에 시간이 오래 걸린다.
- **동적 메모리 관리 (OS의 시스템 콜)**:
  - **`getnode()`**: 새로운 노드를 저장할 메모리를 시스템으로부터 할당받아 그 주소를 반환한다. (만약 메모리가 꽉 찼다면 널 포인터를 반환한다.)
  - **`putnode(i)`**: 주소 $i$에 있던 노드를 지우고, 사용하던 메모리를 다시 시스템에 반납(재활용)한다.

### 3-2. 연결리스트의 4가지 형태 (매우 중요)

**① 단일 연결리스트 (Singly Linked List)**

- 가장 단순한 기본 형태다.
- **노드 구성**: 실제 데이터를 담는 `elem` + 다음 노드의 주소를 가리키는 `next` 링크 1개.
- **특징**: 무조건 한 방향(앞에서 뒤)으로만 이동할 수 있다. 마지막 노드의 `next` 링크는 더 이상 가리킬 곳이 없으므로 **널 링크($\emptyset$, Null)**를 저장한다.
- **접근점**: 리스트의 시작을 알리는 **헤드(Head)** 노드의 주소가 유일한 접근점이다.

**② 이중 연결리스트 (Doubly Linked List)**

- 단일 연결리스트의 단점(뒤로 못 감)을 해결하기 위해 링크를 하나 더 추가한 형태다.
- **노드 구성**: 이전 노드의 주소 `prev` + 데이터 `elem` + 다음 노드의 주소 `next`.
- **특징**: 양방향 순회가 가능하다. 첫 노드의 `prev`와 마지막 노드의 `next`는 널($\emptyset$)이다.
- **접근점**: 처음인 **헤드(Head)**와 끝인 **테일(Tail)** 노드의 주소 두 개를 접근점으로 유지한다.

**③ 원형 연결리스트 (Circular Linked List)**

- 마지막 노드가 널($\emptyset$)을 가리키는 대신, 둥글게 말아서 **다시 첫 번째 노드(Head)**를 가리키게 만든 리스트다.
- **특징**: 어느 노드에서 시작하든 링크를 계속 따라가면 전체 노드를 모두 순회할 수 있다.
- **접근점**: 주로 **테일(Tail)** 노드의 주소를 접근점으로 삼는다. 테일만 알고 있으면 `Tail->next`를 통해 헤드(Head) 노드도 즉시 $O(1)$ 만에 알 수 있기 때문에 양쪽 끝단 접근에 매우 효율적이다.

**④ 헤더 및 트레일러 노드 (더미 노드 / Sentinel Nodes)**

- 연결리스트를 구현할 때 코딩을 아주 편하게 만들어주는 일종의 꼼수(테크닉)다.
- **특징**: 리스트의 맨 앞(Header)과 맨 뒤(Trailer)에 실제 데이터는 들어있지 않은 **가짜 노드(Dummy node)**를 미리 만들어 둔다.
- **사용 목적 (왜 쓰는가?)**:
  이 가짜 노드들이 양끝을 든든하게 막아주기 때문에, 아무리 리스트가 텅 비어있거나 맨 끝에 데이터를 추가/삭제하더라도 **모든 실제 데이터 노드는 무조건 양옆에 앞/뒤 노드를 가지게 된다.** 따라서 빈 리스트이거나 첫 노드를 지울 때 발생하는 특수한 예외 처리(Edge case, `if` 조건문 등)를 전부 생략할 수 있어 코드가 훨씬 깔끔해진다.
